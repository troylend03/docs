## Scheduled Maintenance

We occasionally perform maintenance that may affect availability:

- **Timing:** Usually late night/early morning (US time)
- **Duration:** Typically 15-60 minutes
- **Notification:** Posted on status page in advance
- **Impact:** Service may be unavailable or slow

## During an Outage

If Rallies is down:

### Do:
- Check the status page for updates
- Wait for the issue to be resolved
- Try again in a few minutes

### Don't:
- Repeatedly refresh (can slow recovery)
- Submit support tickets about known outages
- Panicâ€”we're working on it!

## Report an Issue

If you're experiencing problems not shown on the status page:

1. Try basic troubleshooting first:
   - Refresh the page
   - Clear cache
   - Try a different browser
2. Check if others report the same issue (social media, forums)
3. [Contact support](/support/contact) if the issue persists

Include:
- What's happening
- When it started
- Your browser/device
- Screenshots if possible

## Historical Incidents

The status page maintains a history of past incidents:

- View previous outages and their causes

## Uptime Commitment

We strive for high availability:

- **Target:** 99.9% uptime
- **Monitoring:** 24/7 automated monitoring
- **Response:** Team alerted immediately to issues
- **Communication:** Updates posted throughout incidents

## Data During Outages

Your data is safe during outages:

- **Not lost:** Watchlists, portfolios, history preserved
- **Synced on recovery:** Any pending changes sync when service returns
- **Backed up:** Regular backups protect against data loss
